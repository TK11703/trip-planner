# Homepage UI Contract: Branding Refresh

This contract defines the refreshed home page behavior and content expectations.

## Required composition

- Present a large welcome image or image-led hero in the first viewport.
- Keep the primary menu options available through the existing navigation shell.
- Keep recent trips navigation available for signed-in users.
- Provide a clear primary action for planning or creating a trip.
- Provide a clear secondary action for reviewing existing trips or learning how to plan successfully.
- Keep a hint of continuing content visible below the first hero area on typical desktop and mobile viewports.

## Visitor state

- Show the refreshed brand mark and image-led welcome surface.
- Explain trip planning benefits without discussing implementation technology.
- Include helpful planning examples or tips suitable for a new user.
- Provide sign-in and learn-more style actions without requiring trip data.

## Signed-in state with recent trips

- Show the refreshed welcome surface without hiding recent trips.
- Retain direct navigation to create a trip and review recent trips.
- Show recent trips in a branded but scannable presentation.
- Do not replace user trip names or user-entered content even if they contain old terms.

## Signed-in state with no trips

- Show a polished empty state with the refreshed brand language.
- Offer a create-trip action.
- Include a short practical planning tip or example.
- Avoid old explorer/journey wording.

## Transport references

- References to car, train, plane, and boat may appear in icon rows, planning tips, route examples, or subtle hero details.
- Transport references must support the planning story; they must not imply booking, ticketing, or live transit integrations.
- Transport references must remain understandable without color alone.

## Layout and responsiveness

- Desktop may use larger image composition and richer supporting details.
- Tablet and phone layouts must keep actions and recent trips reachable without layout overlap.
- Hero text must remain legible on top of the selected image in both light and dark modes.
- Buttons, menus, and cards must keep stable dimensions and touch-friendly spacing.
